🤖 Claude Code

💡
Full Platform Support

Claude Code has native support for skills, agents, and pipelines. All OMNISKILL features work out of the box.

Target Directory

~/.claude/skills/

Format

Each skill becomes a SKILL.md file in a skill directory with optional resources.

Installation

$ python scripts/install.py --platform claude-code

# Output:
# ✓ Detected Claude Code at ~/.claude/
# ✓ Installing 47 skills...
# ✓ Installing 8 bundles...
# ✓ Installing 12 agents...
# ✓ Installing 4 pipelines...
# ✓ Installation complete!

Features

Feature Support Notes
Skills ✓ Full Native skill format
Bundles ✓ Full Meta-skills work perfectly
Agents ✓ Full Native agent support
Pipelines ✓ Full Full workflow orchestration
Resumability ✓ Full State persistence supported

Platform-Specific Tips

  • Use natural language triggers — Claude Code has excellent semantic matching
  • Pipeline handoffs are seamless — no manual intervention needed
  • Resources can include images and diagrams — Claude can interpret them

💻 GitHub Copilot CLI

💡
Full Platform Support

Copilot CLI has comprehensive skill and agent support with pipeline orchestration.

Target Directory

~/.copilot/skills/

Format

Each skill becomes a SKILL.md file with optional YAML front-matter for metadata.

Installation

$ python scripts/install.py --platform copilot-cli

# Output:
# ✓ Detected Copilot CLI at ~/.copilot/
# ✓ Installing skills with front-matter...
# ✓ Configuring agent bindings...
# ✓ Setting up pipeline triggers...
# ✓ Installation complete!

Features

Feature Support Notes
Skills ✓ Full Front-matter for metadata
Bundles ✓ Full Meta-skills via front-matter
Agents ✓ Full Agent definitions supported
Pipelines ✓ Full Workflow orchestration
Resumability ✓ Full State files in ~/.copilot/

Platform-Specific Tips

  • Trigger keywords are case-insensitive
  • Use ghcs command for skill invocation
  • Pipeline state is saved automatically between steps

⚡ Cursor

⚠️
Skills + Basic Agents

Cursor supports skills and basic agent definitions. Complex pipelines require manual handoffs.

Target Directory

.cursor/rules/ (project-level) or global rules directory

Format

Each skill becomes a .mdc rule file.

Installation

$ # Project-level installation
$ python scripts/install.py --platform cursor --project-local

# OR global installation
$ python scripts/install.py --platform cursor --global

Features

Feature Support Notes
Skills ✓ Full Compiled to .mdc rules
Bundles ○ Partial Meta-skills work, no atomic ops
Agents ○ Partial Basic agent definitions only
Pipelines △ Manual Requires manual step execution
Resumability ✗ No State must be managed manually

Platform-Specific Tips

  • Use project-level rules for project-specific skills
  • Global rules are good for universal utilities
  • Cursor's inline suggestions work well with skill outputs
  • For pipelines, manually invoke each agent in sequence

Platform Override Example

skills/my-skill/overrides/cursor.md
# Cursor-Specific Behavior

## Additional Instructions for Cursor

When running in Cursor:
- Use inline suggestions for code completion
- Leverage Cursor's file navigation for multi-file changes
- Output format optimized for .mdc rules
- Use @-mentions for file references

🌊 Windsurf

⚠️
Skills + Basic Agents

Windsurf supports skills and basic agents. Pipeline orchestration is manual.

Target Directory

.windsurfrules or rules directory

Format

Skills are compiled into Windsurf rule format.

Installation

$ python scripts/install.py --platform windsurf

# Output:
# ✓ Detected Windsurf
# ✓ Compiling skills to Windsurf format...
# ✓ Installing to .windsurfrules
# ✓ Installation complete!

Features

Feature Support Notes
Skills ✓ Full Compiled to Windsurf format
Bundles ○ Partial Skills installed together
Agents ○ Partial Basic personas supported
Pipelines △ Manual Manual step execution
Resumability ✗ No Not supported

Platform-Specific Tips

  • Windsurf has excellent context awareness — skills can be more concise
  • Use Windsurf's native file handling for multi-file operations
  • Override files can optimize for Windsurf's capabilities

🚀 Antigravity

⚠️
Skills + Custom Format

Antigravity uses a unique skill format. OMNISKILL adapts skills accordingly.

Target Directory

.antigravity/skills/

Format

Each skill becomes a skill file in Antigravity's expected format.

Installation

$ python scripts/install.py --platform antigravity

# Output:
# ✓ Detected Antigravity
# ✓ Adapting skills to Antigravity format...
# ✓ Installing to .antigravity/skills/
# ✓ Installation complete!

Features

Feature Support Notes
Skills ✓ Full Adapted to Antigravity format
Bundles ○ Partial Installed as skill sets
Agents ✗ No Not supported yet
Pipelines ✗ No Not supported yet
Resumability ✗ No Not supported

Platform-Specific Tips

  • Antigravity has unique capabilities — use platform overrides to leverage them
  • Skill format is different but adapters handle the conversion
  • Check Antigravity docs for platform-specific features

🌐 Multi-Platform Installation

Install to all detected platforms at once:

$ # Auto-detect all installed platforms
$ python scripts/install.py

# Output:
# ✓ Detected platforms: claude-code, cursor, copilot-cli
# ✓ Installing to Claude Code...
# ✓ Installing to Cursor...
# ✓ Installing to Copilot CLI...
# ✓ All platforms updated!

Target Specific Platforms

$ # Install to specific platforms only
$ python scripts/install.py --platform claude-code --platform cursor

# Install a specific bundle to multiple platforms
$ python scripts/install.py --bundle web-dev-kit --platform claude-code --platform copilot-cli

🔧 Platform Overrides

Skills can have platform-specific behavior via overrides/ directory:

📁 skills/my-skill/overrides/
📄 cursor.md — Cursor-specific additions
📄 windsurf.md — Windsurf-specific additions
📄 antigravity.md — Antigravity-specific additions

Overrides are merged with the base SKILL.md during adapter transformation.

✓
When to Use Overrides
  • Platform has unique capabilities you want to leverage
  • Platform has limitations that require workarounds
  • Output format needs to be different for the platform
  • Platform-specific tool or command usage
