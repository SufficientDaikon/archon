Frequently Asked Questions

Common questions about OMNISKILL's capabilities, installation, and usage.

🌐 General

What platforms does OMNISKILL support? ▼
OMNISKILL supports 5 AI coding assistant platforms:
  • Claude Code — Full support (skills, agents, pipelines)
  • GitHub Copilot CLI — Full support
  • Cursor — Skills + basic agents
  • Windsurf — Skills + basic agents
  • Antigravity — Skills only
See the Platform Guide for detailed capabilities.
Can I use just one bundle instead of everything? ▼
Yes! You can install specific bundles: 
$ python scripts/install.py --bundle web-dev-kit
This installs only the skills in that bundle. You're not required to install everything.
Do I need all 5 platforms installed? ▼
No. You need at least one supported platform, but not all of them. The installer auto-detects what you have and installs only to those platforms. 
$ python scripts/install.py
# Detects: claude-code, cursor
# Installs to those two only
Is OMNISKILL free and open source? ▼
Yes! OMNISKILL is open source under the MIT license. You can use it freely, modify it, and contribute improvements back to the community.

⚡ Skills

How do I create a new skill? ▼
Two ways:
  1. Recommended: Tell your AI assistant "create a new skill for [domain]" to use the skill factory pipeline.
  2. Manual: Copy skills/_template/ to skills/my-skill/ and follow the Creating Skills guide.
My skill's triggers conflict with another skill. What do I do? ▼
Three options:
  • Make triggers more specific: "create React component" instead of "create component"
  • Use patterns: "create * component in React" to be more targeted
  • Create a bundle: Put both skills in a bundle with conflict-resolution rules in bundle.yaml
The validator will warn you about trigger conflicts during validation.
Can a skill inherit from another skill? ▼
Yes! Set extends: parent-skill-name in your manifest.yaml. The child skill inherits the parent's instructions and resources, which you can then override or extend.
manifest.yaml
name: advanced-react
extends: react-best-practices
description: "Extends react-best-practices with advanced patterns"
How do I test my skill? ▼
Add test cases in tests/cases/:
tests/cases/basic.yaml
name: "Basic skill test"
input: "User prompt that should trigger this skill"
expected:
  contains: ["expected output"]
  sections: ["required sections"]
Then run: python scripts/validate.py skills/my-skill

📦 Bundles

What's the difference between a bundle and just installing multiple skills? ▼
Bundles include a meta-skill that:
  • Resolves conflicts when multiple skills match
  • Routes between constituent skills intelligently
  • Provides atomic install/update/uninstall operations
  • Shares resources across all skills in the bundle
Without a bundle, skills operate independently with no coordination.
Can I create a bundle with skills from different authors? ▼
Yes! The bundle's bundle.yaml just references skill names. The skills can be from different authors. Your bundle provides the orchestration layer via the meta-skill.
How do I update a bundle? ▼ 
$ # Update to latest version
$ git pull origin main
$ python scripts/install.py --bundle my-bundle --update

# This updates all constituent skills and the meta-skill

🔄 Pipelines

Can I create custom pipelines? ▼
Yes! Create a YAML file in pipelines/ following the pipeline schema. See Creating Pipelines for a complete guide. 
name: my-pipeline
steps:
  - name: step-one
    agent: first-agent
    input: "user request"
    output: "intermediate result"
  - name: step-two
    agent: second-agent
    input: "output from step:step-one"
    output: "final result"
What happens if a pipeline step fails? ▼
Depends on the step's on-failure setting:
  • halt: Stop the pipeline and report failure
  • loop: Go back to loop-target step and retry (up to max-iterations)
  • skip: Skip this step and continue to the next
  • retry: Retry this same step (up to max-iterations)
Can I resume a pipeline from where it left off? ▼
Yes, if the pipeline has resumable: true: 
$ # Resume from last completed step
$ python scripts/resume-pipeline.py my-pipeline

$ # Resume from specific step
$ python scripts/resume-pipeline.py my-pipeline --from-step implement
State is saved in the state-file specified in the pipeline YAML.
Do pipelines work on all platforms? ▼
  • Claude Code: Full pipeline support with automatic handoffs
  • Copilot CLI: Full pipeline support
  • Cursor: Manual step execution required
  • Windsurf: Manual step execution required
  • Antigravity: Not supported yet
See the Platform Guide for details.

🤝 Contributing

How do I contribute a new skill? ▼
  1. Fork the OMNISKILL repository
  2. Create your skill in skills/my-skill/
  3. Validate it: python scripts/validate.py skills/my-skill
  4. Test it on at least one platform
  5. Submit a pull request
See CONTRIBUTING.md for full guidelines.
Can I contribute to the adapter for my favorite platform? ▼
Absolutely! Adapter improvements are always welcome. Check adapters/ to see how existing adapters work, then submit a PR with your improvements.

If you want to add support for a new platform, open an issue first to discuss the approach.
Where should I report bugs or request features? ▼
Open an issue on the GitHub repository. Use:
  • 🐛 Bug reports: Include platform, skill name, error message
  • ✨ Feature requests: Describe the use case and expected behavior
  • 📖 Documentation: Point out what's unclear or missing

🔧 Troubleshooting

The installer says it can't find any platforms ▼
The installer looks for platform directories:
  • ~/.claude/ for Claude Code
  • ~/.copilot/ for Copilot CLI
  • .cursor/ for Cursor (project-level)
  • .windsurfrules for Windsurf
  • .antigravity/ for Antigravity
Make sure you have at least one AI coding assistant installed and configured.
A skill isn't triggering when I expect it to ▼
  1. Check the skill's triggers in manifest.yaml
  2. Try a more specific phrase that exactly matches a trigger keyword
  3. Check if another skill with higher priority is matching instead
  4. Verify the skill is installed: python scripts/doctor.py
Some platforms have better semantic matching (Claude Code) while others need exact keywords (Cursor).
Validation is failing but I don't understand why ▼
Run validation with verbose output: 
$ python scripts/validate.py skills/my-skill --verbose
This shows detailed error messages including:
  • Which section is missing or malformed
  • Which trigger conflicts with what
  • Which resource file can't be found
Where can I get help? ▼
  • Documentation: Start with the Getting Started guide
  • GitHub Issues: Search existing issues or open a new one
  • Examples: Look at existing skills in skills/ for reference
  • Community: Join the discussions on GitHub
