## How to Use

When generating a DESIGN.md:
1. Identify which archetype fits the project
2. Read the corresponding reference file
3. Extract the relevant token patterns
4. Customize for the specific project

When these examples conflict with project requirements, project requirements win.
When the design brief is ambiguous, these examples serve as defaults.
